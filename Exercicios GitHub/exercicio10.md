# Exercício 10 – Desafio Final

Este repositório foi criado para demonstrar a aplicação de recursos avançados do Markdown no GitHub, cumprindo o desafio proposto no exercício.

---

## 🚀 Demonstração de Recursos Avançados do Markdown

Abaixo estão renderizados **5 elementos diferentes** e avançados suportados pelo GitHub Markdown.

### 1. Detalhes Expansíveis (`<details>` e `<summary>`)
Ideal para ocultar grandes blocos de código ou spoilers, mantendo o README limpo.

<details>
<summary><b>Clique aqui para expandir e ver o segredo! 🤫</b></summary>

> 🎉 Parabéns! Você descobriu como criar uma seção retrátil usando tags HTML dentro do Markdown.
</details>

---

### 2. Texto Riscado (Strikethrough)
Utilizado para indicar tarefas concluídas, correções ou mudanças de planos. 

* ~~Isto é um exemplo de texto riscado antiquado.~~
* Este é o novo padrão atualizado!


---

### 3. Escudos Personalizados (Badges)
Muito utilizados em repositórios profissionais para indicar status do projeto, tecnologias e links úteis.

### 4. Alertas e Notas de Destaque (Admonitions)
Recurso nativo do GitHub para destacar avisos importantes, notas ou dicas visuais usando sintaxe de blockquote com tags específicas.

> [!NOTE]
> Esta é uma nota útil para chamar a atenção do leitor para um detalhe importante do projeto.

> [!WARNING]
> Cuidado! Certifique-se de configurar as variáveis de ambiente antes de rodar a aplicação.
